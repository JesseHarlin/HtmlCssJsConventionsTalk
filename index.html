Nomenclature

  • Html tags and css declarations and properties are lowercased. No uppercase.
  • <!-- Nay --> <A Href="YAY.html"></A> <P CLASS="camelCASE? "></P> <!-- Yay! --> <img src="hawt_drizzle_muffins.png" alt="Tasty Noms" />

Nomenclature

  • Always close tags. The parser will try to help you, but use good form.
  • Namspace your css. The dash has meaning (usually)
  • Html, use double versus single quotes (this iwll affect your js, believe it or not)
  • Use meaningful id and class names. DO not use presenational names
    • /*yes*/ .ui-widget-content /*no*/ .green-button .alt #SDFG-4567

Naming

  • Indentation: Don't mix tabs and spaces. Many guides say use 4, others 2 spaces. Set your editor, and stick to it.
  • Avoid qualifying ID and class names with type selectors Bad: ul#exampleBetter: #example
  • Use shorthand properties where possible. (Css)

On Brevity

  • Use shorthand properties where possible. (Css)
  • No: margin-bottom: 45px; margin-left: auto; margin-right: 1em; /*Mixing measurements = double bad*/ margin-top: 0%;
  • Yes: margin: 0 auto 0 45px; /*or*/ margin: 0 auto; //omit units on zero
  • You cal also express rgba(255,255,255)/ #ffffff as #fff,

On Casing, wording

Use hypens

no: .myClassName

no: .my_class_name

yes: .my-class-name

I've seen a lot of underscores out there for Id's, but rarely if ever, classes.

Every Css prop needs a semicolon.

Typically, there is a space between the selector and props, eg

#yourid { margin-top: 1em; }

Separate multiple selectors with commas

Opinions

Js style is a moving target

There are many wildly differning opinions

Case in point :semicolons

My opinion: Semicolons are GOOD

Best resource. is: Javascript patterns

  • Use Object literal Notation over traditioanl OO notation (constructors)

Yes!

var array = []; var obj = [];

NO:

var array = new Array(); var obj = new Object(); 
                    //clean
                    var obj = {prop1: 1, pro2: [1,2,3]};
                    //dirty
                    var obj = new Object();
                    obj.prop1 = 1;
                    obj.prop2 = new Array();
                    obj.prop2[0] = 1 //and so on.

But when you do, use Capitals for constructors

                // constructor
                    function Waffle() {
                        this.tastes = "yummy";
                    }
                    // a new object
                    var good_morning = new Waffle();
                    console.log(typeof good_morning); // "object"
                    console.log(good_morning.tastes); // "yummy"
                    // antipattern:
                    // forgotten `new`
                    var good_morning = Waffle();
                    console.log(typeof good_morning); // "undefined"
                    console.log(window.tastes); // "yummy"
